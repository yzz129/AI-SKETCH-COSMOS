# 单一 Splat GPU 骨骼蒙皮

## 目标

角色始终使用完整的 `model.splat` 渲染。系统不再把身体、手臂、耳朵、尾巴导出成多个高斯模型，也不在运行时用 CPU 循环改写高斯点。

不可见代理骨架只提供关节、父子层级和蒙皮权重，不参与最终画面，因此不会降低原始 Splat 的外观质量。

## 生成流程

1. TripoSplat 导出完整 `model.splat`。
2. 豆包视觉模型并行识别主体边界、部位轮廓、连接根部和末端。
3. `splat_skinning.py` 用纯 NumPy 将完整高斯中心聚合成不可见的三维体素代理笼，并保留每个原始高斯到代理体素的映射。
4. AI 坐标只提供语义种子；系统会把根关节校正到朝向父级的连接面中心，再从高斯密度拟合三维骨轴。
5. 每个视觉模型识别出的完整手臂、腿、翅膀、鱼鳍或尾巴只生成一个刚性骨，避免同一部位内部被不同变换撕开；耳朵挂到头骨，头骨再挂到身体。只有视觉模型明确返回独立子部位时才允许增加子骨。
6. 从部位末端的前后壳深度包络向根关节逐段传播，完整带回同一肢体的可见与背面高斯，同时拒绝投影重叠的躯干深度层；部位内部使用单骨刚性权重，只在连接关节窄带混合。
7. 刚性核心的每个高斯只允许一个运动骨拥有；只有连接根部的窄带可与身体骨混合，编码仍保留四个 Spark 权重槽位。
8. 权重按 Spark `RGBA16UI` 格式写入 `rig-weights.bin`，清单写入 `rig.json`。

连接处使用窄带连续权重，不存在拆分模型形成的缝隙；部位主体保持刚性，不再整片软拉伸。权重逐点量化后总和严格为 255。

## 运行时

前端立即加载并显示 `model.splat`，不会等待骨架资源。`rig.json` 和约 512 KiB 的权重文件异步到达后，系统为原模型安装 Spark `SplatSkinning`。

每一帧只计算少量骨骼矩阵并更新骨骼纹理，所有高斯的双四元数变形由 GPU 完成。不可见、爆散或距离裁剪的模型不会提交骨骼更新。

## 质量门禁

- 低于置信度阈值的部位不会生成骨骼。
- 三维骨架长度、半径、覆盖率和图像投影一致性必须通过检查。
- 根关节必须位于父级方向的连接面中心，不能落在手、脚、耳朵、嘴或部位自身中心。
- 连接根部只在窄带内平滑混合，远离关节的高斯必须通过刚性核心比例门禁。
- 刚性部位核心允许身体权重降为零；只有连接面的窄带同时保留父骨与子骨权重，避免整片软拉伸。
- 模型数量、权重文件长度和骨骼索引在浏览器安装前再次验证。
- Rig 生成或加载失败时保留完整模型和整体验证过的运动，不影响作品入屏。

## 资产格式

`rig.json` 的生产版本为 `version: 7`，`strategy` 必须为 `gpu-splat-skinning`，`segmentationMethod` 必须为 `distal-depth-track-v1`。`quality.usesProxyVoxelCage`、`quality.usesNarrowJointBands` 和深度轨迹质量指标用于确认新管线已生效；`proxyPreviewUrl` 指向校正后的关节链预览。

`rig-weights.bin` 按原始 `.splat` 顺序保存，每个高斯包含四个小端 `uint16`：高 8 位是骨骼索引，低 8 位是权重。文件长度必须等于 `sourceGaussianCount * 4 * 2`。

历史 `rig-*.splat` 文件仅作为迁移回滚备份存在，当前前后端不会访问这些文件。

# AI Sketch Cosmos

将儿童画、插画或普通图片转化为可观察、可互动、会成长的 3D 粒子生物，并让它们生活在同一片宇宙中的交互式 Web 应用。

项目不只是一个“图片转 3D”演示：作品上传后会经历识别、生成、入场、环游、投喂、升级和自动遭遇等阶段。前端可以先用图片粒子即时展示，配置 TripoSplat 后端后则会生成可从多个角度观看的 Gaussian Splat，并在语义部位 Rig 就绪后获得摆手、迈腿、摆尾等局部动作。

## 功能

- **画作识别**：提取主体形态、颜色、部位、运动倾向和性格特征，为入场姿态与后续行为提供参数
- **2D → 3D 双路径**：前端图片粒子可即时入场；可选 TripoSplat 后端生成高质量 Gaussian Splat 资产
- **新作品聚光入场**：作品准备完成后依次经历 `fly-in → showcase → release → idle`，相机会自动靠近展示并平滑返回
- **宇宙生态场景**：生物在星空、星云、银河和行星之间运行，具有漂浮、旋转、跳动和局部肢体动作
- **直接互动**：支持旋转视角、点击生物爆散重生、点击空白区域投放星光食物，以及按住或拖动画布触发空间坍缩
- **成长与等级**：生物吸收星尘获得经验，等级和经验进度显示在模型旁；战斗胜负也会影响等级
- **自动事件**：生物会自主追逐、逃跑、碰撞、战斗、胜利庆祝、挣脱行星吸引或穿越星系传送门
- **语义部位动画**：后端可为同一个 `.splat` 生成 GPU 蒙皮 Rig，在不拆散主体的情况下驱动手、脚、翅膀、尾巴等部位
- **作品管理**：`/admin` 页面支持查询、筛选、修改、软删除、恢复、永久删除和模型资源下载

## 功能玩法

### 核心体验循环

1. 点击“发布作品”或“发射作品”，选择一张图片。
2. 前端显示识别与生成状态；识别和 3D 生成会尽量并行执行。
3. 基础模型准备完成后，新生物以聚光方式飞入镜头，完成一次近景展示。
4. 生物回到星河轨道，与已有作品共同游动；Rig 尚未完成时不会阻塞首次显示，资源就绪后会自动热加载局部动作。
5. 玩家通过投喂、观察和点击制造互动，系统则持续组织追逐、战斗、行星吸入和传送等自动事件。
6. 经验与战斗结果推动生物升级，让同一件画作从“展示品”逐渐变成具有状态的宇宙生命。

### 操作方式

| 操作 | 结果 |
|------|------|
| 点击控制面板中的发布按钮 | 上传画作，启动识别、存储和 3D 生成流程 |
| 鼠标或触屏拖动场景 | 暂停自动环绕，从不同角度观察星河和 Gaussian 模型 |
| 点击一个空闲生物 | 触发彩色粒子爆散、流星拖尾与重生聚合；聚光或自动事件期间不会重复触发 |
| 点击空白星空 | 在 3D 场景中投放星光食物，附近生物会受到食物吸引 |
| 按住并拖动画布 | 以指针位置为中心触发引力透镜、旋涡、冲击波和色差坍缩效果 |
| 点击“隐藏面板” | 保留沉浸式星河画面；可通过紧凑按钮再次展开 |
| 点击“清空” | 清除当前前端场景中的作品状态 |
| 点击“全屏” | 进入或退出沉浸式全屏展示 |

### 生物成长与自动生态

- 生物初始为 `LV 0`，吸收系统投放的星尘后积累经验；升级所需经验随等级增长。
- 等级徽章会显示当前等级和经验条，便于在星河中比较不同作品的成长进度。
- 生物会根据距离、等级和当前状态产生追逐或逃跑意图。遭遇进入战斗后，胜者升级并庆祝，败者降级但最低不会低于 `LV 0`。
- 行星可以捕获靠近的生物，表现为吸入、挣扎、受困和逃脱；传送门可把被追逐的生物送往另一个星系位置。
- 自动事件具有冷却和互斥条件。正在聚光、爆散或参与其他事件的生物不会被随意打断，从而避免状态叠加造成动作冲突。

> TripoSplat 负责把单张图片生成 3D Gaussian 资产；“游动、投喂、成长、战斗、传送和特效”由 AI Sketch Cosmos 的前端行为系统负责。两者组合后才形成完整玩法。

## 从图片到星河生命

```text
上传图片
  ├─ 视觉识别 → 形态 / 部位 / 色彩 / 行为特征
  ├─ 即时路径 → 图片采样 → 本地粒子生物
  └─ 3D 路径 → TripoSplat → model.splat
                              ├─ 前端立即显示完整模型
                              └─ 后台语义部位分析 → rig.json + rig-weights.bin
                                                   → GPU 局部动作热加载

作品入场 → 聚光展示 → 返回轨道 → 投喂成长 / 自动遭遇 / 点击特效
```

系统遵循“基础资产优先显示、增强资源异步到达”的原则。即使 TripoSplat、部位识别或 Rig 加载失败，也会尽量保留图片粒子或完整静态 `.splat`，而不是让作品无法进入星河。

## 技术栈

| 层 | 技术 |
|------|------|
| 前端框架 | React 19 + TypeScript |
| 3D 渲染 | Three.js + @react-three/fiber + @react-three/drei |
| 后处理 | EffectComposer + UnrealBloomPass + 自定义 GLSL Shader |
| 3D 模型 | @sparkjsdev/spark (Gaussian Splat 渲染) |
| 状态管理 | Zustand |
| 2D Canvas | 原生 Canvas 2D (划屏拖尾特效) |
| AI 识别 | 火山方舟 Ark API |
| 后端 (可选) | FastAPI + TripoSplat + PyTorch |
| 构建 | Vite |

## 特效系统

### 划屏拖尾 (TouchTrailCanvas)
2D Canvas 叠加层，鼠标/触摸移动时产生彩虹粒子拖尾：柔光外发光 + 亮核心，AdditiveBlending 叠加。

### 点击爆开流星拖尾 (SplatCreatureModel 粒子代理)
点击 3D 生物时触发：
- **420 个粒子**从模型表面采样，球面均匀向外扩散
- 片段着色器采用**高斯衰减**函数，实现柔和流星效果
- 流星形态：微小高亮头部 + 锥形渐变宽度的发光轨迹
- 颜色覆盖**全彩虹光谱**，高饱和度（98%~100%）
- 粒子随时间指数衰减（`exp(-dist * 2.8)`），符合物理光照衰减规律

### 全屏塌陷后处理 (Effects CollapsePass)
长按/拖拽时触发引力透镜效果：径向拉拽 + 旋涡扭曲 + 冲击波环 + 色差偏移。

## 后端 (可选)

参见 [backend/README.md](backend/README.md)

启用 TripoSplat 3D 生成需设置环境变量：
```bash
VITE_TRIPOSPLAT_ENABLED=true
VITE_TRIPOSPLAT_API_BASE=http://127.0.0.1:8000
```

后端提供作品任务、历史作品、静态模型资源和管理接口。主页会同步后端中可用的历史作品；管理入口为 `/admin`。完整的部署方式、模型权重和推理参数见 [部署指南](docs/DEPLOYMENT.md)。
